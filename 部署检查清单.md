### 4. 配置服务器

**静态文件配置**（Nginx示例）：
```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /path/to/dist;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 启用gzip压缩（可选）
    gzip on;
    gzip_types application/javascript text/css application/json;
}
```

**静态文件配置**（Apache示例）：
```apache
<VirtualHost *:80>
    ServerName yourdomain.com
    DocumentRoot /path/to/dist

    <Directory /path/to/dist>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted

        # SPA fallback
        RewriteEngine On
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteRule ^ index.html [L]
    </Directory>
</VirtualHost>
```

## 常见部署问题

### 1. 缓存问题

**症状**：
- 页面显示旧版本
- Console报错找不到bundle文件

**解决方案**：
- 清除浏览器缓存（强制刷新）
- 删除服务器上的旧bundle文件
- 在bundle文件名中添加版本号（Vite自动处理）

---

### 2. MIME类型错误

**症状**：
- `Failed to load module script`
- `MIME type ('text/plain') is not executable`

**解决方案**：
- 配置服务器正确返回 `.js` 文件的 MIME类型
- Nginx/Apache默认配置通常已正确
- 添加配置（如果缺失）：
  ```nginx
  types {
      application/javascript js;
  }
  ```

---

### 3. 跨域问题

**症状**：
- `CORS policy: No 'Access-Control-Allow-Origin'`

**解决方案**：
- 确保所有资源在同一域名下
- 或配置CORS（如果需要跨域）：
  ```nginx
  add_header Access-Control-Allow-Origin *;
  ```

---

### 4. PWA配置问题

**症状**：
- `manifest.json` 加载失败
- Service Worker注册失败

**解决方案**：
- 确保 `manifest.json` 在正确位置
- 检查manifest路径：
  ```html
  <link rel="manifest" href="/assets/manifest-BbxDU7OK.json">
  ```
- 确保服务器返回正确的manifest文件

## 开发环境 vs 生产环境

### 开发环境（npm run dev）

**特点**：
- 本地开发服务器
- 热更新（HMR）
- 未压缩代码
- 无bundle文件名哈希

### 生产环境（npm run build）

**特点**：
- 压缩优化代码
- Bundle文件名哈希（缓存控制）
- 无热更新
- 需要服务器部署

## 自动化部署

### GitHub Actions示例

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Setup Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '18'

      - name: Install dependencies
        run: npm install

      - name: Build
        run: npm run build

      - name: Deploy to server
        uses: easingthemes/ssh-deploy@main
        with:
          SSH_PRIVATE_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
          REMOTE_USER: user
          REMOTE_HOST: server.com
          SOURCE: dist/
          TARGET: /path/to/webroot
```

## 性能优化建议

### 1. 启用gzip压缩

**Nginx配置**：
```nginx
gzip on;
gzip_comp_level 6;
gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
gzip_min_length 1000;
```

**效果**：
- Bundle大小：155 KB → 44 KB（gzip）
- 减少70%传输大小

---

### 2. 启用HTTP/2

**Nginx配置**：
```nginx
listen 443 ssl http2;
```

**效果**：
- 多路复用，提升加载速度
- 减少连接开销

---

### 3. 启用CDN（可选）

**方案**：
- 使用CDN加速静态资源
- Bundle文件部署到CDN
- 更新HTML中的资源路径
